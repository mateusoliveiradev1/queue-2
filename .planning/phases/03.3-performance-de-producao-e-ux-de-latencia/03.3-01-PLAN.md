<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser metric payload -> performance endpoint | Client-provided metrics are untrusted and must be allowlisted/redacted. |
| Server action/route timings -> logs | Logs must preserve diagnostic value without leaking secrets or duo-private payloads. |
| Production credentials -> baseline evidence | E2E credentials are process-only inputs and must never be written into artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.3-01 | Information Disclosure | Web Vitals endpoint and server timing logs | mitigate | Accept only metric names, stable route/action labels and numeric values; reject arbitrary strings and sensitive keys. |
| T-03.3-02 | Tampering | Client-supplied metrics | mitigate | Treat metrics as observational only; never use them for authorization, rewards or product state. |
| T-03.3-03 | Repudiation | Baseline artifact | mitigate | Record environment type, route list, command status and missing env vars explicitly. |
| T-03.3-04 | Denial of Service | Metrics endpoint | mitigate | Keep payload small, no-store, no DB fan-out in v1 telemetry ingestion, and optional rate limiting if abuse is observed. |
</threat_model>
