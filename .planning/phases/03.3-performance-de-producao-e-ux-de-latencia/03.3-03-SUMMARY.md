## Decisions Made

- Existing redirect-based server actions remain the fallback path for native/no-JS form submission.
- Enhanced server actions reuse the same validation, membership checks and domain use cases, then return only `ok`, `state`, optional `redirectTo` and optional `liveId`.
- Confirmed UI copy is never shown before the server action resolves.
- Reduced-motion support uses static `/2` feedback marks instead of adding new animated spinners.
