## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Optimized SQL -> RLS policies | Faster query shapes must still execute under transaction-local app user context. |
| Query-plan artifacts -> private data | EXPLAIN output and query review must not include private duo payloads or secrets. |
| Timing wrappers -> business modules | Measurement code cannot become a new place for business rules. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03.3-06 | Elevation of Privilege | Optimized repository queries | mitigate | Keep `withAppUserTransaction`, membership lookups and RLS; never accept client duo ids. |
| T-03.3-07 | Tampering | Data-changing EXPLAIN review | mitigate | Use rollback-wrapped transactions or static plans; do not persist benchmark mutations. |
