<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Optimistic client state -> authoritative server state | Local UI feedback is never the source of truth for duo state. |
| Enhanced action result -> native form fallback | Server validation and authorization must be identical for enhanced and fallback paths. |
| Repeated clicks -> domain mutation | Double-submit prevention and idempotent server behavior must stop duplicate user-visible effects. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.3-09 | Spoofing | Client optimistic state | mitigate | Use local copy only as pending/syncing; server result controls confirmed state. |
| T-03.3-10 | Tampering | Enhanced mutation inputs | mitigate | Keep server-side validation for UUIDs, statuses, return paths and membership. |
| T-03.3-11 | Repudiation | Mutation failures | mitigate | Preserve server telemetry and explicit failure/retry copy instead of silent rollback. |
| T-03.3-12 | Denial of Service | Double submits | mitigate | Disable pending controls, add tests for duplicate clicks and rely on existing idempotent database shapes. |
</threat_model>
