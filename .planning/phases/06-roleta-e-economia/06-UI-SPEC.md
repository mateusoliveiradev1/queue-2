## Typography

New roulette UI uses exactly four sizes and two weights: 400 and 900.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 900 | 1.35 |
| Heading | 20px | 900 | 1.2 |
| Display | 40px | 900 | 0.95 |

Usage rules:

- Body text uses Inter Tight and sentence case.
- Labels, counters, buttons and economy chips use JetBrains Mono, uppercase, and 14px.
- Section headings use Archivo Black at 20px, uppercase.
- The result title may use the 40px display role, but it must wrap before resizing and must not overlap the cover or actions on mobile.
- Do not use negative letter spacing. Letter spacing is `0`.

## Phase Surface Contract

`/app/roleta` is a playable authenticated app surface, not a landing page. The first viewport must show one of these states immediately: blocked pool, ready roulette, resumable reveal, pending result invitation or history-backed empty trust state.

Layout rules:

- Add `Roleta` to authenticated navigation with `currentPage="roleta"`.
- Navigation order: `Fila`, `Catalogo`, `Descobrir`, `Biblioteca`, `Roleta`, `Conquistas`, `Desafios`, `Dupla`, `Perfil`.
- Desktop uses the existing app shell with a wide reel band spanning the content column. Side metadata may sit beside controls, but the reel itself is not placed inside a small card.
- Mobile uses a horizontal full-bleed reel with the central pointer fixed over the visual center and controls below it.
- Mobile navigation must not overlap or clip nine items. Use a horizontally scrollable min-width track or another tested pattern that keeps labels readable and touch targets at least 44px.
- Use cards only for result, blocked state, history rows and replacement choices. Do not nest cards inside the reel band.

Reel contract:

- Render exactly 60 visual cover slots for a reveal round.
- Visual repeats are allowed when the eligible pool has fewer than 60 games.
- The selected result is a single server-persisted game; the reel must not imply the client selected it.
- Cover slots use stable `aspect-ratio: 3 / 4`, object-fit cover, and reserved dimensions before images load.
- Decorative reel slots are `aria-hidden`; announce the reel as one region and announce only the persisted result.
- The central pointer uses `RoulettePointer` with `tone="primary"` in ready/revealing states and may use `tone="accent"` only for boosted/special result states.

## Interaction Contract

State machine:

| State | Visual Contract | Interaction Contract |
|-------|-----------------|----------------------|
| Loading/resuming | `/2` loading mark or skeleton slots, no generic spinner | Rehydrate from server state; never show a new spin prompt if an active/pending round exists |
| Blocked pool | Dashed `empty-state`, no fake reel | Show CTAs to `Biblioteca`, `Descobrir` and `Catalogo`; explain three-game minimum |
| Ready | Reel preview, balance, boost toggle, pity copy, audio control | `Sortear da fila` starts a server action; button shows `aria-busy` within 100ms |
| Persisting | Controls disabled, action feedback visible | No result shown until server returns the persisted round |
| Revealing | 5.5s horizontal reel motion with central pointer | Use `cubic-bezier(.15,.85,.25,1)`; expose polite live status |
| Reduced motion | Three staged status panels, no long moving reel | Same persisted result, no loss of information or action |
| Pending invitation | Result panel with cover, rarity seal and commitment copy | Primary action locks as Principal; secondary action discards; replay is clearly replay only |
| Replacement required | List current three `Jogando` games as explicit choices | Nothing pauses automatically; user chooses one replacement or cancels |
| Locked | Success feedback and route handoff to `/app` | Highlight new Principal on dashboard using the existing pointer/primary accent |
| Discarded | Calm confirmation and history update | Game remains Wishlist/Pausado; no refund for a persisted reveal |
| History | Compact rows with rarity, boost/pity usage and outcome | No Hall replay, stats dashboard or broad storytelling in this phase |

Audio contract:

- Audio is opt-in after explicit interaction only.
- Respect existing duo audio preference before playing tick, drumroll or fanfare.
- Provide one visible mute toggle near the main control group.
- Muting audio must not disable the visual reveal or reduced-motion reveal.
- Audio status changes use polite `aria-live` copy and no toast unless the user explicitly toggled.

Motion contract:

- Normal reveal lasts approximately 5.5 seconds.
- Tick cadence can grow heavier near the pointer, but avoid casino/slot-machine exaggeration.
- Legendary particles are contained inside the result/reel area and last no longer than the reveal completion moment.
- Scanlines are allowed only inside the roulette reel/result experience at low opacity; do not add global scanlines.
- `prefers-reduced-motion: reduce` disables reel travel, particles, pulsing and long transitions.

## Accessibility Contract

- All actionable controls meet 44px minimum target size and visible `var(--focus-ring)`.
- Keyboard users can start, mute, replay, lock, choose replacement, cancel and discard without pointer-only gestures.
- The 60-cover reel is not exposed as 60 focusable items.
- The result panel receives focus after reveal completion, unless focus is inside a confirmation dialog.
- `aria-live="polite"` announces persisted, revealing and locked states; failed sync uses assertive feedback.
- The destructive discard confirmation uses explicit copy and requires a deliberate confirm action.
- Rarity must be communicated by text seal plus border color, never color alone.
- Reduced-motion users receive the staged reveal by default and can complete every action without timed movement.
- Mobile bottom navigation, reel controls and result actions must not sit under `env(safe-area-inset-bottom)`.

---

## Visual Anti-Patterns

Do not use:

- Casino chips, slot levers, roulette-table green, money piles or betting metaphors.
- Full-page violet/purple glow, decorative bokeh, generic arcade neon or confetti storms.
- A tiny decorative reel card that makes the ritual secondary.
- Individual totals, member comparison, leaderboards or solo reward language.
- Result copy that sounds like a command. The result is a commitment invitation.
- Disabled buttons that look active, or history/replay copy that implies a new draw.
