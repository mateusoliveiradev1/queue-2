<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser duration/percent -> server | Client durations and percentages may be forged or extreme. |
| Confirmation -> XP/progress | A single member cannot finalize a session or duplicate XP. |
| Catalog estimate -> progress UI | Source/freshness must remain honest and not imply unsourced precision. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Spoofing | Session confirmation | mitigate | Current user only; DB unique confirmation per member; partner confirmation required. |
| T-04-03-02 | Tampering | Session duration/percent | mitigate | Server validates bounds and computes live elapsed from timestamps. |
| T-04-03-03 | Repudiation | Chapter/session awards | mitigate | Store actor IDs, timestamps, award keys and domain events. |
| T-04-03-04 | Information Disclosure | Game detail progress | mitigate | RLS-scoped reads and no client-supplied `duoId`. |
| T-04-03-05 | Elevation of Privilege | XP award | mitigate | Award only in confirmed transaction with unique award key. |
</threat_model>
