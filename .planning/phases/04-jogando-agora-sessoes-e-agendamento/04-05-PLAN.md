<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser push subscription -> server | Endpoint/key material is untrusted and sensitive. |
| Cron invocation -> reminder processor | Requests may be forged, duplicate or overlapping. |
| Notification polling -> private duo state | Pending actions and failures are private duo data. |
| `/app/dupla` settings -> product push state | Opt-out must not break schedules or confirmations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Spoofing | Cron endpoint | mitigate | Require exact `Authorization: Bearer ${CRON_SECRET}` and no redirects. |
| T-04-05-02 | Tampering | Attendance/schedule edit | mitigate | Validate duo/game ownership; reset confirmations in transaction on significant edit. |
| T-04-05-03 | Repudiation | Reminder sends/failures | mitigate | Store job attempts, notification items and domain/audit events. |
| T-04-05-04 | Information Disclosure | Push endpoints | mitigate | Server-only storage, endpoint hash redaction and no browser secret exposure. |
| T-04-05-05 | Denial of Service | Notification polling/jobs | mitigate | Batch due jobs, bound polling reads, indexes on unread/due state. |
| T-04-05-06 | Elevation of Privilege | Notification actions | mitigate | RLS and server authorization for each actionable item. |
</threat_model>
