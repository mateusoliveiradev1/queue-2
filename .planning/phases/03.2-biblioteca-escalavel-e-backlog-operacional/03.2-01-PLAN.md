<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL params -> library use case | Route params are untrusted and must be normalized before repository reads. |
| Library route -> repository | The route must consume public application contracts, not query database or own queue policy. |
| Runtime DB user -> duo-scoped rows | Bounded reads still run with transaction-local user identity and forced RLS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.2-01 | Tampering | `LibraryQueueInput` | mitigate | Normalize view, status, sort, page, limit, query and platform in application/domain before repository reads. |
| T-03.2-02 | Information Disclosure | `getQueue` SQL | mitigate | Keep reads inside `withAppUserTransaction`; never accept client-supplied duo id; rely on membership and RLS. |
| T-03.2-03 | Denial of Service | Biblioteca reads | mitigate | Require bounded `LIMIT/OFFSET`, clamped page sizes and no full-library hydration in the new route path. |
| T-03.2-04 | Elevation of Privilege | Archive contract | mitigate | Archive is read/policy contract only in this plan; terminal status mutation remains blocked until Phase 4 confirmation. |
</threat_model>
