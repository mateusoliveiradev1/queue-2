<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Presentation controls -> server action | Buttons and sheets submit existing server actions; client UI does not authorize status changes. |
| Filter form -> route params | GET controls produce URL params that Plan 02 parser and Plan 01 use case normalize. |
| Component imports -> module boundary | Presentation components cannot import library infrastructure or other module internals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.2-09 | Tampering | Status action forms | mitigate | Preserve hidden form inputs and existing server action validation; never infer status mutation from client-only state. |
| T-03.2-10 | Spoofing | Secondary action menu/sheet | mitigate | Hide terminal future actions entirely; only active statuses are submitted. |
| T-03.2-11 | Denial of Service | Mobile controls | mitigate | Use compact filters and single primary action so controls do not overlap or make the page unusable. |
| T-03.2-12 | Information Disclosure | Component imports | mitigate | Keep components presentation-only and run `pnpm check:architecture`. |
</threat_model>
