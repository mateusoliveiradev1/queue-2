| Boundary | Description |
|----------|-------------|
| URL params -> server read | Search params are user-controlled and must be normalized before calling the library use case. |
| Route composition -> business rules | `page.tsx` may choose layout but not redefine active/archive/status rules. |
| Archive count -> user-visible UI | Archive count cannot become a dead or misleading route surface before Phase 4 creates real archived items/behavior. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03.2-06 | Information Disclosure | `/app/biblioteca` | mitigate | Derive user id from `requireVerifiedSession`; call `getLibraryQueue` only for current user. |

</threat_model>
