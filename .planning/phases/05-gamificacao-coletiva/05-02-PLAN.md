<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Play/Discovery confirmed fact -> gamification | Only server-confirmed facts are accepted; source modules cannot bypass gamification policy. |
| Client mutation -> reward result | Browser-supplied amount, level, streak, quest progress or achievement IDs are ignored. |
| Ledger -> projection | Projection update must stay transactional and rebuildable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | XP amount/source | mitigate | Map source facts to server-side policy amounts and reject client amounts. |
| T-05-02-02 | Repudiation | Critical rewards | mitigate | Ledger rows carry source type/id, actor and metadata; failures abort the confirming transaction. |
| T-05-02-03 | Denial of Service | Duplicate facts | mitigate | Unique ledger/source keys and idempotent returns prevent duplicated effects. |
| T-05-02-04 | Elevation of Privilege | Cross-module internals | mitigate | Public contracts, ports and architecture checks block deep imports. |
| T-05-02-05 | Information Disclosure | Reward summaries | mitigate | Reward reads stay duo-scoped and RLS-backed. |
</threat_model>
