<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> gamification surfaces | Client cannot supply XP, level, streak, quest progress or achievement unlock authority. |
| Play/Discovery -> gamification | Source modules pass confirmed facts through public contracts/events only. |
| Gamification -> Postgres | Runtime uses RLS and least privilege; migrator applies schema changes. |
| Worker -> scheduled jobs | Rotation/streak jobs may duplicate or overlap and must be idempotent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Spoofing | XP facts | mitigate | Apply rewards only from server-side confirmed facts and current duo membership. |
| T-05-01-02 | Tampering | Ledger/projections | mitigate | Append-only ledger, projection transactions, adjustment records and rebuild tests. |
| T-05-01-03 | Repudiation | Rewards/unlocks | mitigate | Store source type/id, actor, timestamps, metadata and domain events. |
| T-05-01-04 | Information Disclosure | Achievement/quest/streak rows | mitigate | Forced RLS and cross-duo tests for every duo-scoped table. |
| T-05-01-05 | Denial of Service | Quest/streak jobs | mitigate | Due indexes, batch limits, `FOR UPDATE SKIP LOCKED` and retry caps. |
| T-05-01-06 | Elevation of Privilege | Cross-module imports | mitigate | Public contracts only and architecture check coverage. |
</threat_model>
