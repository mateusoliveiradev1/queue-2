## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cron -> maintenance route | Only CRON_SECRET-authorized calls may run jobs. |
| Worker -> duo timezone windows | Job code must compute windows server-side from duo timezone. |
| User actions -> quest progress | Only authoritative confirmed facts progress quests. |
| Challenge UI -> state | UI cannot create completion, failure or reward state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | Spoofing | Maintenance endpoint | mitigate | Require `Authorization: Bearer ${CRON_SECRET}` and source tests. |
| T-05-05-02 | Tampering | Quest windows | mitigate | Unique cycle keys and server-side timezone calculations. |
| T-05-05-03 | Repudiation | Streak freeze consumption | mitigate | Append streak/freezing events with source job/fact metadata. |
| T-05-05-04 | Denial of Service | Job scans | mitigate | Batch limits, due indexes and `FOR UPDATE SKIP LOCKED`. |
| T-05-05-05 | Information Disclosure | Challenge history | mitigate | No visible failure marks for incomplete challenges. |
</threat_model>
