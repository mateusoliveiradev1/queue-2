## Decisions Made

- Discovery application use cases own card read-model construction because these use cases combine catalog cards, read state, recommendation facts and library action state before UI rendering.
- The architecture checker applies same-module layer direction to static imports/exports. Dynamic application wrappers that load server-only infrastructure remain allowed because Phase 3 documented that runtime dependency pattern.
- Catalog had equivalent existing layer inversions exposed by the new checker, so those pure helpers and record shapes were moved rather than weakening the global architecture rule.
