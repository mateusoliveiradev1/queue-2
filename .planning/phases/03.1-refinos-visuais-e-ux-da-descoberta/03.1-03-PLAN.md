<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client pending state -> server result | UI feedback is provisional until server actions redirect with authoritative status. |
| Search input -> search API | Autocomplete input remains bounded by the existing authenticated route and rate limiter. |
| Match celebration -> library mutation | Celebration cannot imply or perform library movement without explicit handoff form submit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-07 | Tampering | `DiscoveryDeck` and `DiscoveryCard` pending state | mitigate | Keep hidden form values and server actions as the only mutation path; local reaction state is visual only. |
| T-03.1-08 | Information Disclosure | `DiscoverySearch` overlay | mitigate | Fetch only same-origin `/api/discovery/search` with existing auth/rate-limit path; do not expose RAWG or database secrets. |
| T-03.1-09 | Repudiation | Match handoff state | mitigate | Require explicit Wishlist/Jogando/Pausado form submit and keep match-only state separate from library/roulette eligibility. |
| T-03.1-10 | Denial of Service | Motion and overlays | mitigate | Reduced-motion fallback, keyboard operation, focus-visible styles and no hidden unreachable controls. |
</threat_model>
