## Shared Patterns

- Authenticated routes must call `requireVerifiedSession()` and redirect pairing/naming states before rendering private data.
- Discovery route files must import domain capabilities through `apps/web/src/modules/discovery/index.ts`, not module internals.
- Server actions own session validation and redirect/status mapping; React components receive actions as props and submit forms.
- Client Components can animate and track pending UI state, but final match/decision/library state stays server-authoritative.
- RAWG/source/freshness metadata remains visible through typed view models and text-first UI; it must not become decorative-only.
- E2E skips must list missing variable names. A skipped browser fixture is not passing evidence.
