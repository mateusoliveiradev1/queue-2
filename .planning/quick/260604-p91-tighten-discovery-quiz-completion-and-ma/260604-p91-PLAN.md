# Quick Task 260604-p91: Tighten Discovery Quiz Completion And Match Live Flow

## Goal

Fix Phase 03.1 flow drift on `/app/descobrir`: the Quiz should not keep asking the current member after their mood answers are saved, and Match Live should have a clear inactive/active state instead of duplicate start controls that look inert.

## Product Decisions

- Keep Discovery filters available outside Match Live because `DISC-08` requires general discovery filters.
- Remove duplicate Live start entry from the orbit controls. The top control navigates to the Live panel; the panel owns the start action.
- Fetch the latest active Live session even when the URL has no `live=` parameter.
- Hide the Quiz tray and shortcut once the current member has answered the current round.
- Use the local `impeccable` product/polish guidance: earned familiarity, no invented affordances, clear states, no decorative controls.

## Tasks

1. Add a Discovery mood quiz status read model in the application/repository layer.
2. Use the status on `/app/descobrir` to omit the Quiz shortcut/tray after the current user has answered.
3. Make Match Live stateful: one start action when inactive, active status when live exists, no duplicate start button.
4. Update tests for the completed Quiz and active/inactive Live flows.
5. Run focused verification.
